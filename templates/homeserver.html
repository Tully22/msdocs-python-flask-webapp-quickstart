Building a homelab

I built a homelab using my old PC!

Posted by Harry Tull on November 15, 2021

I think of the cloud computing revolution to have really kicked in around 2011/12. Some may argue earlier as by 2010, Amazon, Google, Microsoft had all launched cloud divisions, some may argue even earlier! depending on the context i.e. AWS first launching in 2006 and Google Docs was available the same year (this service definetly contributed to some of my university group projects!). However I think of this revolution happening as cloud services were adopted by organisations, and I started to see the buzzwords in advertised job roles, eventually the idea of using public and private clouds together was not seen as a risk.

Hybrid clouds enabled organisations to better customise their implementations and integrate cloud services into their business while still utilising infrastructure they had invested in for legacy services. In addition the ability to better and easier forecast their IT OpEx as they only pay for what they use.

In 2011, iCloud was released by Apple, enabling retail consumers to begin using cloud storage in their daily lives, a couple years down the line and now most entity's have Office365 (a nice little push from covid-19 has sped this up too).

In this blog post I will be creating a homeserver, or homelab, to house and test my projects/solutions using my old PC, eventually I would like to invest in a more powerful homeserver, but for now this is fit for purpose.

Objective: Build a homelab that I can use to house my personal projects on using an old computer

Getting Started

Well it's been sat in the garage since I left for Africa, so to keep things short and sweet, I gave her a thorough clean, fired her up, backed up any old files on my external HD (data hoarder alert!), I noticed it in fact only has 8GB of RAM (it's also got a basic Mobo, i7 3rd Gen CPU and 2 HDD's)..I mean we're on a budget here but 8GB in today's world is pathetic! With that noted I quickly used my Mum's Amazon Prime Account and ordered me 2x8GB RAM sticks and 2 SSD's. While I was waiting for them to be posted I wiped my machine clean and formatted all drives.

..This image is pre tidy up (Mum's hairdryer and a few cotton buds to clean gunk off the fans) and removal of the CD Drive, I did also buy a new bracket to fasten my SSD's...

Okay, so after some thought I have decided I want to have the power to switch between OS's, create and kill Virtual Machine's at my will, spin up Docker containers as and when for my project's, all the while on piece of metal that is acting as a server running 24/7 (RIP mum's electric bill)...the answer is..Proxmox! ..Right so what is and why Proxmox? Proxmox Virtual Environment is based on the Debian Linux distribution for server virtualization. It allows me to install different OS's (Windows, Linux and others) on a single computer or a cluster built by grouping computers together. It also has a neat little web accessible GUI, so once it is installed up and running I can access and boss it via my laptop on the LAN and leave my server headless. Secondly it's open source so well within budget!

How did I do this?

Well.. in short, I downloaded a few ISO's (for later), but first made a bootable USB loaded Proxmox on, plugged it in my machine then installed it.

1. Downloaded the ProxMox VE ISO file

2. Downloaded a tool that will convert my usb stick into a bootable one (Rufus)

3. Found an old USB stick of 16GB and flashed the Proxmox ISO to it

4. Inserted the USB into my old PC (monitor connected) and installed Proxmox from USB, I specified my new 120GB SSD as the target hard disk

5. Created a hostname during install and then checked my LAN for a free IP address to assign to it (for remote access on the laptop later)

6. Rebooted after install

7. Used the IP address to access it via my laptop (I have BT HomeHub, to stop it from assigning a new address I logged on to the router and specified it to remain static, there is a better solution for this but that's another adventure for another day).

System Architecture

As stated, I've taken my old PC, spent a hundred quid giving it some more fire power and below is my architecture in mind, one physical machine with multiple VM's and ability to run Docker on it. Why? For instance, my application (future project!) may need to interact with a database that resides in a one of my VM's. Provided that the right networking is in place, the application can interact with the database on the VM.

Creating a VM

For my first VM, I want to use Ubuntu Server and then install Docker and run an image that has MS SQL Server..(Why? I had this set-up once for a client, it's pretty easy to understand, lightweight and perfect for a future project I have in mind). Once this is done this will be another computer available on my network, I tried to explain it to my Step-Dad (who looked increasingly p***ed off as I messed about with his phone cabling)..using the film Inception as a metaphor (dream within a dream..computer within a computer)..anyway turns out he hasn't seen the film, that's 5 minutes I'll never get back.

I should mention, I grabbed a couple more ISO's for later (Ubuntu Desktop and Windows) stuck them on another USB, added the USB to the back of my server and then in Proxmox mounted it as a drive..now I have a little place to store my ISO's and access in Proxmox.

..Obviously it's called tullserver

Installing Docker and MS SQL Server

So now my Ubuntu Server is up and running (with it's own IP address), I can interact with it via SSH in PowerShell, or directly through the console Proxmox gives me in the web GUI. I did it using SSH.

Command 'sudo systemctl status docker' lets me know its working..next pull the image I want here using 'docker pull mcr.microsoft.com/mssql/server:2019-latest'

Magic..remotely (using my laptop) connected to my SQL Server that is sat in my container that resides on my Ubuntu Server..that is a VM..I made using Proxmox..that sits on my physical server..that sits in my Mum's living room!

Conclusion

I was able to do this in one evening, well if you include the Amazon Prime order installations and cleaning my PC..2 evenings, but due to me having a strong understanding in hardware, my target infrastructure and system architecture I made light work of this. I now have the ability to host cloud databases to use for my projects/applications, older HDD's for NAS to access on my LAN (I told you I data hoard) all the while having the ability for performance monitoring and backup via Proxmox. As mentioned in future I might invest in a new build homelab but for now this is perfect for what I want to do!
